# Categorização de Skills (Pessoal / Eduzz) — User Stories

## Autor (Furihata)
- Como autor, quero manter todas as minhas skills em um único repositório (`lp-skills`) para não administrar dois repos, dois sites e dois mecanismos de distribuição.
- Como autor, quero que cada skill pertença a um bucket (`personal` ou `eduzz`) refletido na estrutura de pastas para organizar o source por contexto.
- Como autor, quero trazer as skills do `labzz-skillzz` (afl, jira, notion-pull, notion-push, make-dev) sanitizadas para o repo público para descontinuar o `labzz-skillzz` com segurança.
- Como autor, quero que o `/jira` seja genérico (sem `AV-` nem credencial exibida) e que a especificidade Eduzz (`AV-*`) viva no `/afl` para reaproveitar o `/jira` fora do contexto Eduzz.

## Dev usuário do site
- Como dev usuário, quero ver no site a categoria de cada skill (Pessoal / Eduzz) para entender o contexto antes de instalar.
- Como dev usuário, quero filtrar o catálogo por categoria para focar só nas skills relevantes ao meu uso.
- Como dev usuário, quero que o prompt de instalação gerado continue funcionando independentemente do bucket em que a skill está para instalar sem fricção.

## Colega da Eduzz (potencial)
- Como colega da Eduzz, quero encontrar as skills de trabalho (jira, afl, notion-*) marcadas como "Eduzz" no mesmo catálogo para localizá-las rapidamente.
